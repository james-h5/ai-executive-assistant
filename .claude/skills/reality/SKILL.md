# Reality Skill

**Trigger:** Run this skill when James asks anything like "reality check", "be honest with me", "am I on track", "what's the fastest path", "what should I actually be doing", or similar.

## Step 1: Load Context
Before saying anything, read:
- `context/current-priorities.md`
- `context/goals.md`
- `context/work.md`
- `context/me.md`

Check `logs/daily/` for the most recent daily log if it exists — look for patterns of what's been worked on vs avoided.

If ClickUp MCP is available, pull open tasks to see what's actually in flight.

## Step 2: Diagnose Honestly

Assess the current state against the top goal: **$10k/month location-independent income**.

Ask yourself:
- What is the single biggest bottleneck right now?
- Is James working on the bottleneck, or working around it?
- What is being avoided that matters most?
- What feels productive but isn't moving the needle?
- How close is he actually to the next concrete milestone?

## Step 3: Deliver the Reality Check

Structure the output like this:

---

### Reality Check — [Date]

**Where you're trying to get:**
[One sentence on the goal — the life, not the metric]

**The honest diagnosis:**
[2–4 sentences. What's actually happening right now. No softening. If effort is scattered, say so. If the bottleneck is obvious, name it.]

**The bottleneck:**
[One thing. The single constraint that, if removed, would unlock the most progress. Be specific.]

**What I'd do this week if I were you:**
[3–5 bullets. Concrete actions. Not "work on consulting" — "call two electricians you know and offer to run the system for free for 30 days". Specific enough to act on immediately.]

**What to deprioritise:**
[2–3 bullets. Things that feel useful but aren't the bottleneck. Name them plainly.]

**The uncomfortable truth:**
[One paragraph. The thing James probably already knows but hasn't fully acted on. Deliver it straight — not harsh, but no softening either.]

---

## Output Rules
- No padding, no preamble — start with the diagnosis
- Casual and direct tone — like a trusted friend who knows the plan and isn't afraid to tell the truth
- Be specific: name the actual tasks, the actual people, the actual bottleneck
- Don't invent problems that aren't there — if he's on track, say so clearly
- The goal is clarity and momentum, not motivation-speak
- Never end with generic encouragement — end with a clear direction
- Keep the whole output readable in under 2 minutes
