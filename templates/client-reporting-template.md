# Client Reporting Template

Send monthly, timed to when the ongoing fee is billed — the report is the justification for the
recurring charge, so it should land right around then, not weeks later.

---

## Report Structure

```
# Monthly Report — [Business Name]
Period: [Month Year] | Prepared by James Heathcote

## Headline
[1-2 sentences — the single most useful stat this month, e.g. "The system handled 34 enquiries
this month, 9 of which came in after hours or while you were on a job."]

## By The Numbers
| Metric | This Month | Last Month |
|---|---|---|
| Total enquiries handled | | |
| New leads (qualified) | | |
| Existing customers recognized | | |
| Average response time | | |
| After-hours/on-the-job enquiries caught | | |
| Follow-ups sent (no reply within 24hr) | | |

## Notable Saves
[1-3 short examples of a specific enquiry the system caught that would likely have been missed
otherwise — this is the "proof it's working" section, more persuasive than the raw numbers]

## Anything That Needs Your Input
[Any wording tweaks noticed, new FAQ that came up repeatedly, edge case worth discussing]

## Next Month
[Anything planned - no action needed if nothing's changing]
```

## Notes
- If a month is quiet (few enquiries), say so plainly rather than padding the report — credibility
  compounds, and a trades business will notice a slow month before you tell them.
- Where actual data isn't available yet (e.g. no analytics wired up), be upfront that figures are
  estimated, and prioritize getting real tracking in place for the next report.
